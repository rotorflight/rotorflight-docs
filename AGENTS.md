# AGENTS.md — Rotorflight Docs

## Project Overview

Docusaurus 3 static site for [Rotorflight](https://www.rotorflight.org/) — an open-source RC helicopter flight controller. Built with React 18, TypeScript, and MDX. Node ≥ 20 required.

## Architecture

* **`docs/`** — Next (dev) version docs in Markdown/MDX. This becomes the next release version.
* **`versioned_docs/`** — Frozen docs per release (`version-2.0.0/`, `version-2.1.0/`, `version-2.2.0/`). Managed by Docusaurus versioning; do not manually create version folders.
* **`versions.json`** — Declares released versions `["2.2.0", "2.1.0", "2.0.0"]`.
* **`sidebars.ts`** — Defines all sidebar trees. Uses `autogenerated` for simple sections (download, Tuning, Manufacturers, Contributing) and explicit item lists for the main `tutorialSidebar`.
* **`docusaurus.config.ts`** — Site config, plugin setup, client redirects, navbar, footer. Loads `.env` via `dotenv/config`. Uses `FASTER=true` env var to enable SWC/Rspack experimental build.
* **`src/theme/`** — Swizzled Docusaurus theme components (Navbar, ColorModeToggle, DocCardList, SearchBar). These wrap originals; keep wrapper pattern when modifying.
* **`src/components/`** — Custom React components (e.g., `GearCalc/` — gear ratio calculators using JSON data from `src/resources/`).
* **`src/pages/`** — Standalone pages: `index.tsx` (homepage with video hero), `donate.mdx`, `markdown-page.md`.
* **`announcement/`** — Blog/announcement posts (routed to `/announcement`).
* **`config/docEditorStatus.json`** — Consumed by the external DocsEditor app; rarely edited.

## Key Commands

```bash
npm run start          # Dev server at localhost:3000
npm run build          # Production build → /build
npm run serve          # Serve production build locally
npm run typecheck      # TypeScript check (tsc --noEmit)
npm run prettier:fix   # Auto-format all files
npm run spellcheck:all # Run cspell across entire project
```

## Pre-commit Hooks (Husky + lint-staged)

On commit, `lint-staged` runs automatically:

* **JS/TS/JSON/CSS/HTML**: Prettier check + cspell
* **MD/MDX**: `remark --output` (lint + auto-fix) + Prettier check + cspell

## Docs Conventions

* Every doc needs YAML front-matter with at least `sidebar_position` (numeric, increments of 10) to control ordering. Example: `sidebar_position: 80`.
* Images for docs go in `docs/img/` (next version) or `versioned_docs/version-X.Y.Z/img/`.
* Internal links use relative paths: `[text](./controllers/index.md)`. Cross-version links use absolute: `/docs/2.0.0/Wiki/...`.
* Redirects for moved/renamed pages are defined in `docusaurus.config.ts` under `@docusaurus/plugin-client-redirects`. The `createRedirects` function auto-generates version-based redirects.

## Formatting & Linting

* **Prettier** (`.prettierrc.ts`): 80 char width, 2-space indent, double quotes, trailing commas, LF line endings.
* **Remark** (`.remarkrc.mjs`): Uses `remark-preset-lint-recommended` with MDX and directive support.
* **cspell** (`cspell.json`): Custom dictionary in `project-words.txt` — add domain-specific terms there (not in `cspell.json`). SVGs, YMLs, and `build/` are ignored.

## Adding a New Doc Page

1. Create `.md` file under `docs/<section>/` with front-matter (`sidebar_position`, `sidebar_label`).
2. If the section uses explicit sidebar items (like `tutorialSidebar`), add the doc ID to `sidebars.ts`.
3. Sections using `autogenerated` (Tuning, Manufacturers, Contributing, download) only need `sidebar_position` in front-matter.

## Versioning a Release

Docusaurus `docs:version` command snapshots `docs/` → `versioned_docs/version-X.Y.Z/` and updates `versions.json`. After versioning, the `docs/` folder becomes the next dev version.

## Environment Variables

Copy `example.env` → `.env`. Key vars:

* `FASTER=true` — Enables SWC/Rspack build (significantly faster)
* `URL` — Site URL (defaults to `https://github.com/`)
* `BASE_PATH` — Base path (defaults to `/`)

## Plugins

* **`@docusaurus/plugin-client-redirects`** — URL redirects for legacy v2.0 paths and version aliases
* **`@signalwire/docusaurus-plugin-llms-txt`** — Generates `/llms.txt` for LLM consumption
* **`docusaurus-lunr-search`** — Client-side full-text search
* **`@docusaurus/theme-mermaid`** — Mermaid diagram support in Markdown (use ` ```mermaid ` code blocks)
